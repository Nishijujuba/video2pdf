# Generate artifact instances at the earliest valid checkpoint

Schema definitions and artifact paths can be known when a run starts, while many artifact instances require evidence that appears later. Pyramid reports require reviewed input fingerprints, compile reports require an actual compile, and the Acceptance Report Skeleton requires final artifact fingerprints plus rendered-page coverage. Precreating these instances during initialization would produce misleading or immediately stale evidence.

## Considered Options

- Create every fixed JSON file during initialization: rejected because evidence-bearing fields cannot be valid before their inputs exist.
- Let agents author required JSON files when prompted: rejected because agents can guess schema shape, omit structural fields, or bind reports to stale artifacts.
- Create artifacts ad hoc without an initialization declaration: rejected because missing outputs and generators remain undiscoverable until a late workflow failure.

## Decision

Run Initialization creates the versioned Video Output Scaffold, the Video Workflow Run Record, and an Artifact Plan. The Artifact Plan declares every expected fixed artifact, governing schema, authoritative generator, planned path, dependencies, and Earliest Valid Generation Checkpoint.

The Video Workflow Kernel invokes the declared generator when that checkpoint becomes ready. Structural and provenance fields are script-owned. Semantic agents or reviewers may supply only the judgment fields explicitly assigned by the contract. Generated templates are fail-closed until those fields are completed and validated.

Pyramid Gate reports are generated after their exact outline, section, or integrated document inputs exist. The final compile report is generated by the guarded final compile. The allowed-artifact manifest, rendered-page evidence, and Acceptance Report Skeleton are generated during delivery readiness after final artifacts exist. Under ADRs 0028 and 0030, Text and Visual Reviewers produce separate bounded Judgment Patches and the Final Acceptance Provider materializes the final Acceptance Report v2.

## Consequences

Every contract and generator is visible from initialization, while live evidence remains current and fingerprint-bound. Agents no longer invent report structure. The scaffold inventory, Artifact Plan schema, generator registry, conditional artifacts, and regeneration or invalidation behavior require follow-up decisions.
