# Triage Labels

The engineering skills use five canonical triage roles. This file maps those roles to GitHub labels in this repository.

| Canonical role | GitHub label | Meaning |
| --- | --- | --- |
| `needs-triage` | `needs-triage` | Maintainer evaluation is required |
| `needs-info` | `needs-info` | Additional information is required |
| `ready-for-agent` | `ready-for-agent` | Human-approved and ready for agent implementation |
| `ready-for-human` | `ready-for-human` | Human implementation or intervention is required |
| `wontfix` | `wontfix` | The work will not be actioned |

A label describes workflow state. It does not replace the human publication gate.
