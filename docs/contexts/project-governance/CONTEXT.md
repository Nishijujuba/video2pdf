# Project Governance Context

Status: active.

This context owns the language for current Project 2.0 planning and publication authority. Project 1.0 planning language belongs to the Legacy Planning Archive and cannot govern new planning work.

## Project 2.0 Spec

A human-approved GitHub planning item that defines the problem, solution, user stories, implementation decisions, verification seams, and scope for a Project 2.0 change.

## Implementation Ticket

A human-approved, context-sized vertical slice derived from a Project 2.0 Spec. It owns independently verifiable acceptance criteria and explicit blocking relationships for one unit of implementation work.

## Human Publication Gate

The required human approval checkpoint before an agent publishes or materially changes a Project 2.0 Spec, Implementation Ticket, milestone assignment, containment relationship, or dependency relationship.

## Legacy Planning Archive

The completed Project 1.0 planning record retained as read-only historical evidence. It may inform current decisions and cannot receive new planning items, status changes, or execution authority.
