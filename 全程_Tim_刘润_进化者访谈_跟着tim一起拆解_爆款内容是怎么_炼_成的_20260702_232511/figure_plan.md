# Figure Plan

## 使用原则

- 原始视频帧主要承担“访谈节点锚定”和“关键原话证明”的作用。画面本身多为访谈镜头，信息量集中在字幕和章节栏，适合每章开头或转场处使用。
- 机制性内容优先使用生成示意图。原因很直接：爆款特质、团队协作、OKR、商业化和使命迭代都属于关系模型，用截图讲会显得松散，用图讲更清楚。
- 所有视频帧插入正文时建议用 `width=0.70\linewidth,height=0.30\textheight,keepaspectratio` 起步。原始帧带有下方章节栏和大字幕，放得过大会挤压正文。
- 生成图优先使用 `.pdf`；同名 `.png` 是视觉检查和后备素材。

## 注意事项

- ASR 对 “HKR / HKR2” 缩写有噪声；四个组成要素在字幕里很清楚：快乐、知识、共鸣、节奏。正文应解释要素，谨慎展开缩写来源。
- 原始帧多为访谈镜头，且带有底部章节导航。正文中应让视频帧承担证据锚点，避免连续堆放访谈截图。
- 若 writer 需要更高分辨率访谈帧，可从 `source_video.mp4` 在相同时间点重抽。当前 figure agent 未新增抽帧，只复制了已检查的 raw frames。
- 生成图都有同名 `.png` 后备文件，位于 `figures/generated/`。
