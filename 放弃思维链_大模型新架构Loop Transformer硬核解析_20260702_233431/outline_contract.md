# 《让模型在隐藏状态里循环思考：Loop Transformer 的机制、证据与工程边界》大纲契约

## 0. 金字塔顶层答案

本文的中心教学判断：

Loop Transformer 的核心贡献，是把测试时计算从外部文本 token 轨迹转移到模型内部的隐藏状态递归。它通过复用少量 Transformer 层，让同一个共享模块在多次递归中逐步更新表示，从而用“有效深度”换取更紧凑的参数量与潜在推理空间。这个方向有真实吸引力：它能把思维链中的“写出中间步骤、回读、再更新”压缩成隐藏状态上的迭代；它也带来新的硬问题：递归稳定性、隐藏推理是否真的发生、每个 token 的计算量如何分配、KV cache 如何控制、以及缺少显式推理轨迹后怎样监督。

因此，最终 PDF 的主线应当明确告诉读者：Loop Transformer 尚未证明可以全面取代标准 Transformer 或 Chain-of-Thought。更准确的定位是：它是一条把参数共享、递归深度、动态系统稳定性、自适应计算和部署约束绑在一起的研究路线。它最可能先在小模型、边缘部署、蒸馏、合成数据生成或同参数量模型竞争中展现价值；在大规模通用推理场景中，表达能力、延迟、吞吐、监督信号和缓存开销仍然是绕不开的限制。

结构选择理由：正文应按“为什么需要内部递归、它怎样运行、有什么实验证据、怎样稳定与加速、最终边界在哪里”的逻辑推进。这个顺序优先建立读者的判断框架，再填入技术细节，避免把字幕改写成逐分钟流水账。

## 4. 术语表

| 术语 | 英文 | 讲义中的统一解释 |
|---|---|---|
| 思维链 | Chain-of-Thought, CoT | 模型通过生成中间文本 token 来分步推理。可类比人在纸上写草稿：每写一步，就把草稿重新读回上下文。 |
| 测试时计算 | test-time compute | 推理阶段额外投入的计算量。CoT 用更多 token 扩展计算；Loop Transformer 用更多递归步扩展计算。 |
| token | token | 模型读写的离散文本单位。CoT 需要把内部想法压成 token，再从 token 还原为隐藏表示。 |
| 隐藏状态 | hidden state | 模型内部的连续向量表示，承载当前 token 和上下文的中间信息。讲义可把它类比成模型脑内的工作记忆。 |
| 残差流 | residual stream | Transformer 中信息逐层累积和传递的主通道。循环架构里，它成为每次递归更新的状态变量。 |
| Loop Transformer | looped Transformer, recurrent-depth Transformer | 复用同一组或少数组 Transformer 层，多次更新隐藏状态的架构。重点是共享权重与递归深度。 |
| 递归深度 | recurrent depth | 同一模块被重复应用的次数。它类似“思考轮数”，但发生在隐藏状态空间中。 |
| 多跳推理 | multi-hop reasoning | 需要多个中间事实或关系链才能得出答案的问题，例如先找第 44 任美国总统，再找其配偶。 |
| Grokking / 阶段性泛化 | grokking / staged generalization | 训练中从记忆、分布内泛化到系统化组合的阶段变化。正文应解释为“模型从背答案走向学会规则”。 |
| 动态系统 | dynamical system | 用状态更新规则研究递归过程的视角。循环模型可写成反复应用同一个更新函数。 |
| 固定点 | fixed point | 递归后状态趋于稳定的位置，满足近似 \(h_{t+1}\approx h_t\)。类比反复搅动后水面趋于平静。 |
| 周期轨迹 | cycle / periodic orbit | 隐藏状态在若干个稳定状态之间循环访问。它说明系统有规律地振荡，而非随机漂移。 |
| PCA | Principal Component Analysis | 把高维隐藏状态投影到二维或三维主方向，用于观察递归轨迹。 |
| MoR | Mixture-of-Recursions | 让不同 token 按需使用不同递归深度的架构思想，用路由器分配计算。 |
| 路由器 | router | 决定某个 token 需要多少递归步，或在每步是否继续处理的轻量决策模块。 |
| token-choice routing | token 选择路由 | 视频语境中指在循环开始前为 token 预分配递归深度桶的方案。若原论文术语不同，需复核后统一。 |
| expert-choice routing | 专家选择路由 | 视频语境中指递归过程中逐步决定继续或退出的方案。需和原论文术语核对。 |
| KV cache | key-value cache | 自回归解码时保存每层 attention 的 key/value 张量，以减少重复计算，但会消耗内存和带宽。 |
| 递归感知缓存 | recursion-aware cache | 只为仍活跃的 token 存储当前递归步所需 KV，随深度增加减少活跃 token 和计算。 |
| 递归 KV 共享 | recurrent KV sharing | 复用较早递归步的 KV，节省内存，但可能使用较旧表示，带来质量折损。 |
| 表达能力 | expressiveness | 模型函数族能表示多少复杂变换。共享同一模块会形成参数约束，独立深层堆栈通常更自由。 |
| 归纳偏置 | inductive bias | 架构天然鼓励模型学习某类模式的倾向。递归鼓励迭代更新，独立深层鼓励分层变换。 |

## 5. 符号表与公式约定

| 符号 | 含义 | 写作约定 |
|---|---|---|
| \(x_i\) | 第 \(i\) 个 token 的输入表示 | 用于解释 token 级别计算路径 |
| \(h_t\) | 第 \(t\) 次递归后的隐藏状态 | \(t=0\) 表示进入循环前的状态 |
| \(h_{i,t}\) | 第 \(i\) 个 token 在第 \(t\) 次递归后的隐藏状态 | 用于 MoR 与 token 级路由 |
| \(F_\theta\) | 共享的 Transformer 递归模块 | \(\theta\) 在不同递归步共享 |
| \(R\) | 最大递归步数 | 可类比模型允许的最大内部思考轮数 |
| \(d_i\) | token \(i\) 被分配的递归深度 | MoR 中体现按需计算 |
| \(g_\phi\) | 路由器函数 | \(\phi\) 是路由器参数 |
| \(a_{i,t}\) | token \(i\) 在第 \(t\) 步是否继续递归的决策 | 可取 continue / exit 或二值 mask |
| \(K,V\) | attention 的 key 和 value 缓存 | 用于解释 KV cache 内存压力 |
| \(m_{i,t}\) | token \(i\) 在第 \(t\) 步是否仍活跃 | 递归感知缓存只保留活跃 token 的 KV |
| \(L\) | 训练损失 | 稳定性和训练曲线中使用 |
| \(\lVert h_t\rVert\) | 隐藏状态范数 | 用于解释状态爆炸和归一化 |

核心公式建议：

1. 循环更新的基本形式：

$$
h_{t+1}=F_\theta(h_t, x)
$$

若强调残差流：

$$
h_{t+1}=\mathrm{Norm}\bigl(h_t + F_\theta(h_t, x)\bigr)
$$

讲解顺序：先说明 \(h_t\) 是模型当前内部表示，\(F_\theta\) 是同一个共享模块；再解释每次递归让模型在连续空间里更新表示；最后说明归一化用于抑制状态范数失控。

2. 固定点与周期：

$$
h^\star \approx F_\theta(h^\star, x)
$$

$$
h_{t+p}\approx h_t
$$

前者表示状态收敛到稳定位置，后者表示状态以周期 \(p\) 重复访问相似区域。

3. MoR 的预分配深度：

$$
d_i=g_\phi(h_{i,0}),\quad d_i\in\{1,\ldots,R\}
$$

4. MoR 的逐步退出：

$$
a_{i,t}=g_\phi(h_{i,t}),\quad a_{i,t}\in\{\mathrm{continue},\mathrm{exit}\}
$$

5. 自适应计算量直觉：

$$
C_{\mathrm{MoR}}\propto \sum_i d_i,\qquad C_{\mathrm{fixed}}\propto nR
$$

其中 \(n\) 是 token 数。正文应强调：MoR 的节省来自简单 token 提前退出，复杂 token 获得更多递归。

6. 朴素递归 KV cache 的压力：

$$
M_{\mathrm{naive}}\propto nR d_{\mathrm{model}}
$$

递归感知缓存可写成：

$$
M_{\mathrm{aware}}\propto \sum_{t=1}^{R} n_t d_{\mathrm{model}},\quad n_{t+1}\le n_t
$$

其中 \(n_t\) 是第 \(t\) 次递归仍活跃的 token 数。

- 00:13:09--00:13:20：朴素递归模型虽然共享参数，仍可能为每个递归深度维护独立 KV cache。
- 00:13:25--00:13:50：递归感知缓存只存储仍活跃 token 的 KV，更深递归处理更少 token。
- 00:13:50--00:14:06：递归 KV 共享复用初始递归的缓存，降低跨深度复制。
- 00:14:06--00:14:35：准确性与内存的权衡：递归感知缓存更新鲜、更准、更耗内存；递归共享更省内存、可能降低质量。

本章必须加入 \(M_{\mathrm{naive}}\)、\(M_{\mathrm{aware}}\) 的直观公式，并把 “参数少” 与 “运行时内存流量少” 区分清楚。

### section_08.tex：边界判断：表达能力、真实深度、监督信号与部署位置

时间范围：00:14:37--00:18:05。

教学目标：给出最终判断，避免把 Loop Transformer 写成单向度的架构胜利。

必须覆盖：

- 00:14:48--00:15:17：表达能力问题。独立更深或更宽的模型拥有更强自由度；在内存充足时，标准 Transformer 可通过增加参数胜出。
- 00:15:17--00:15:31：扩展定律历史上更偏向增加参数量，重复利用计算需要证明其系统价值。
- 00:15:31--00:16:10：递归深度不同于真实独立深层架构。共享函数依赖演化隐藏状态诱导差异化行为。
- 00:16:10--00:16:29：多跳与外推结果仍在受控环境中，大规模真实任务收益尚不明确。
- 00:16:34--00:17:23：CoT 的显式文本轨迹提供监督、筛选、强化和模仿的抓手；隐藏状态递归缺少自然中间标签。
- 00:17:23--00:18:05：潜在应用：同参数量模型竞争、合成数据生成、蒸馏、边缘部署、小模型；大规模服务需考虑延迟与吞吐。

本章最终小结应形成三点判断：

1. Loop Transformer 的优势来自“用计算换有效深度，用共享参数换紧凑模型”。
2. 它的研究价值取决于稳定性、自适应路由、KV cache 和训练监督能否一起成立。
3. 它近期更像特定约束下的工程与研究选项，而非大规模 LLM 架构的默认替代路径。

## 8. 写作约束

1. 文章必须使用中文；首次出现核心英文术语时保留英文原词，如“循环 Transformer（Loop Transformer）”。
2. 每章开头先给本章结论，再展开机制、证据、限制和小结。禁止把字幕按时间顺序平铺。
3. 每个主要章节必须以 `\subsection{本章小结}` 收束，说明读者应带走的判断。
4. 数学公式写作顺序固定为：直觉解释、展示公式、逐项解释符号、说明公式在本章中的作用。
5. 对视频中的实证结果保持证据边界。多跳表现、三阶段泛化、PCA 轨迹、routing 准确率和缓存收益都应附时间来源或论文核验来源。
6. 图像必须服务教学。截图只在解释具体机制、曲线、结构或数值时使用；低信息量过场画面应跳过。
7. 赞助与频道推广不进入正文教学主线。可以在“来源说明”中简短记录其存在。
8. “Claude Mythos” 相关内容只可写成视频中的猜测背景。不得作为技术事实或产品事实。
9. 讲义应适度加入背景知识框：PCA、动态系统、KV cache、归纳偏置、扩展定律。背景框应帮助理解主线，不能替代主线。
10. 对比表达应具体说明机制差异，避免空泛评判。例如比较 CoT 与 Loop Transformer 时，应落到 token 轨迹、隐藏状态、监督信号和运行成本。
